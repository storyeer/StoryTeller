### 1. 🏅 审计大模型课题组：基于微调的财务异常与工程合规模型增强
*大模型微调 (Fine-tuning) · 领域专用模型 (Domain-specific LLM) · 提示词工程 (Prompt Engineering)*
>**项目时间**: 2024.03 - 2025.03

| 🎯 业务目标 | 🛠 核心技术链路 | 📊 核心数据产出 | 🚀 最终模型收益 |
| :--- | :--- | :--- | :--- |
| 构建垂类审计大模型 | `QLoRA 微调`、`TF-IDF 信息抽取` | 交付 **1 万+** 极度纯净规范化微调集 | 成功训练出具备极速法律答疑与文案合规起草能力的 **“审计专属问答 AI”** |

* **[核心工作一]：审计法规与案例的高质量结构化数据集构建**
  * **非结构化信息抽取**：针对海量、零散的审计法律法规与历史罚单，通过构建解析脚本精准提取出**【法规名称】、【出台时间】、【核心要求内容】**及**【处罚措施】**等高维特征，将生语料成功转化为机器可读的 JSON 规范化格式。

### 3. ⚡ 面向速率受限 API 的高性能动态异步调用框架
*高并发工程 · 分布式动态限流 · 自适应背压控制 (Backpressure)*
>**项目时间**: 2025.01 - 2025.05 （**核心算法架构已申报国家发明专利，实审中**）

| 🎯 攻坚痛点 | 🛠 核心纯软算法突破 | ⚡ 性能飞跃指征 | 💡 知识产权成果 |
| :--- | :--- | :--- | :--- |
| **高并发雪崩与配额浪费倒挂** | `动态背压闭环` + `柔性自愈引擎` | API 配额利用率突破至 **100%** | 第一发明人，专利在审 |

* **基于反馈的动态背压传导 (Backpressure)**：彻底摒弃传统固定休眠等待策略。利用拦截器实时解析 HTTP 协议层的 `X-RateLimit` 速率字段，构建动态并发许可计算器。结合平滑因子（α）与目标 P95 延迟联合驱动自动调参，将下游限流信号精准转化为系统级的硬性并发约束与等待时长，实现自适应“背压”传导，使整体吞吐量相较传统轮询拉升幅度达 **408% ~ 676%**。
* **面向 429 熔断的自适应配额重分配**：针对大模型 API 极易频发触发的 `HTTP 429` 速率限制错误，设计了精确恢复机制。在拦截到错误瞬间，对超载的 API 密钥实施即时物理冻结，并基于成功率等权重将冻结配额动态重分配给该池其余可用密钥。此举实现了全局可用资源的极限榨取与防雪崩级保护。
* **面向 50x 死锁的退避与轻量级探测自愈**：针对极易崩溃的 `HTTP 503/504` 服务端宕机困局，坚决规避盲目且无效的高频重试。引入“指数退避算法 + 随机抖动 (Jitter)”缓解并发风暴，并在长窗口的退避期内植入极低频（1-5s）的轻量级 HTTP HEAD 异步健康状态探测。一旦感知服务复苏即瞬时夺回执行机会，使核心故障的**自愈恢复耗时直接骤降 97%（自 200~300ms 断崖式跌落至 9ms）**。

<details>
<summary>💡 <b>展开查看：基于反馈控制的并发许可派发公式</b></summary>
<br>
<blockquote>
核心创新在于将请求并发数（N）转化为受控变量。系统依据计算公式 <code>N_new = N_old × [1 + α·(1 - R_used/R_total) + β·(L_target - L_actual)]</code>，实时嗅探当前时间窗口内的 <b>剩余配额占比</b> 与 <b>P95 响应延迟偏差</b>。<br><br>这套类似 PID 的精妙控制回路，使得每一次调度产生的等待时间都与下游拥塞程度强绑定。在保障 SLO 的前提下，它能驱动系统总吞吐量始终温柔而稳定地逼近 API 服务器允许的理论极值，是实现配额 100% 榨取的灵魂引擎。
</blockquote>
