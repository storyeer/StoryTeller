## 🚀 核心研发经历与产出

### 1. 🏅 审计大模型课题组：基于微调的财务异常与工程合规模型增强
*大模型微调 (Fine-tuning) · 领域专用模型 (Domain-specific LLM) · 提示词工程 (Prompt Engineering)*
>**项目时间**: 2024.03 - 2025.03

| 🎯 业务目标 | 🛠 核心技术链路 | 📊 核心数据产出 | 🚀 最终模型收益 |
| :--- | :--- | :--- | :--- |
| 构建垂类审计大模型 | `QLoRA 微调`、`TF-IDF 信息抽取` | 交付 **1 万+** 极度纯净规范化微调集 | 审计风险靶向识别率由 **50% 跃升至 82%** |

* **[核心工作一]：审计法规与案例的高质量结构化数据集构建**
  * **非结构化信息抽取**：针对海量、零散的审计法律法规与历史罚单，通过构建解析脚本精准提取出**【法规名称】、【出台时间】、【核心要求内容】**及**【处罚措施】**等高维特征，将生语料成功转化为机器可读的 JSON 规范化格式。
  * **智能降噪与去重**：设计基于 TF-IDF 提取稀疏向量并结合余弦相似度的轻量化去重算法，将异构审计语料重复率由 15% 降至 5%，交付 **1 万+** 极度纯净的指令微调集用于下游任务。

* **[核心工作二]：Qwen 基座模型微调实战与风险识别落地**
  * **提示词重构与 QLoRA 调优**：将清洗出的法规特征结合**定制化审计 Prompt 模板**（如 CoT 思维链注入），构建为指令微调集注入 Qwen (2/2.5/3) 大模型。设计并亲自执行了 48 组正交对比实验，探索并锁定了特定垂类场景下的最优学习率退火路径与批次大小。
  * **业务落地成效**：使得模型在真实业务场景下的审计违规风险有效识别率，由未学习规范化数据集前的**50%（基线），破局跃升至专家级的 82%**，同时生成的分析报告逻辑完整度达 70%。

### 2. 🧩 基于 LLM 的智能语义分块与增强检索 RAG 框架
*检索增强生成 (RAG) · 混合大模型推理架构 · 智能文档解析切割*
>**项目时间**: 2025.03 - 2025.08

| 🎯 攻坚痛点 | 🛠 双模态解耦架构 | ⚡ 被打破的瓶颈 | 💡 核心防护成果 |
| :--- | :--- | :--- | :--- |
| **超长法律文本建库** | `DeepSeek(分块)` + `Qwen3(检索)` | 放弃传统 Token 暴力定长截断截断 | 极大缓解大模型长文本的 **幻觉与注意力漂移** |

* **击破截断瓶颈**：独立研发突破传统 Token 暴力定长截断的新型 RAG 框架，实现基于上下文特征的“按语义边界智能分块”。
* **双模态异构调度**：采用 **DeepSeek (负责结构化解析分块) + Qwen3 (负责下游检索生成)** 的解耦架构协作，提升系统扩展性。

| :--- | :--- | :--- | :--- |
